1.1. Application Project Wizard

The application project wizard is used to create new application projects containing domain objects and supporting repositories, factories and other services.

To invoke the wizard, use: File>New...>Project...>Naked Objects>Application Project

The wizard is based in large part on Eclipse's own plug-in project wizard; all projects created by the NO IDE are OSGi projects. A slight refinement is that on the second page the Id field is greyed out, defaulting to the project name. This ensures that the bundle Id in the manifest (META-INF/MANIFEST.MF) is the same as the project name in the .project file.

When the wizard completes, the workbench is placed into the Naked Objects perspective. The Naked Objects nature is also attached on the project, which enables the project builder and the refactoring support.

The classpath of the application project (as defined in the manifest) is minimally dependent on the Naked Objects application library and a very small subset of 3rd party libraries (namely javax.persistence for the @Entity annotation, JUnit and Apache log4j).

As well as setting up the skeleton of the project, the wizard also creates some domain classes and services for an example project. As the tutorial shows, the 3 project wizards work together to build up a complete (albeit very small) and launchable domain application.

Finally, note that as well as the icons needed for the example project, a set of additional icons are also copied (under src/main/resources/images). Developers are free to deploy these icons into production, but only for when running on an NOA-compliant framework.
