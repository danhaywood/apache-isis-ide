6. Annotation Views

The annotation views allow annotations from the Naked Objects application library to be applied to properties, collections, actions, action parameters and also the object itself. They can also be used as a way of navigating through the object. When the annotation is applied the code is automatically updated. Conversely, if the code is updated directly then the annotation view is updated.

Each view consists of a table, with the columns representing the annotations (or sometimes, members of annotations) and the rows representing the features (properties, collections etc.):

  • Marker annotations that have no members - such as @Optional - are shown using a checkbox

  • Single-valued annotations where the member has the special name of value() - such as @Named(value) - are shown using a single column.

    For value() members that specify enumerated types, such as @Disabled(When), these are shown as drop-downs.

  • Complex annotations with multiple members - such as @MemberOrder - are shown using multiple columns.

    Updating any of the columns wil cause that annotation to be present; the annotation is only removed if all of the member values are cleared.

The standard shortcut key bindings for views is the a combination alt+shift+Q, ?. If you pause after alt+shift+Q, then Eclipse will list all the commands available. The annotation views follow this convention, and are bound to alt+shift+Q, n where n=1..5. However if you wish you can change these using Windows>Preferences...>General>Keys.

The views display a lot of useful information about the object. A good tip is to maximise the views (using ctrl+M) while working on them. This is particularly helpful for the actions view.
