3. Create a new Client Project

The role of a client project is to both hold the tests and to run a Naked Objects application (as a set of locally-held launch configuration definitions). In both cases a fixture set is used; it is often helpful to share the fixture set between the application and the tests. The Client Project Wizard creates a sample test and launch configurations for the same 3-class domain example of the Application Project Wizard and the Fixture Project Wizard.

As before use File>New>Project...>Naked Objects to list the NO IDE's project wizards, but this time select the Client Project wizard and then press Next:

The first page of the wizard is similar to that of the Fixture Project Wizard, in that it requires the Application project to be specified. In addition, the Fixture project must also be specified (both will be referenced in the classpath). Press Next:

On the second page of the wizard, enter the client project's Project name. This should have been derived from application project name given on the previous page. Then press Next:

On the third page, adjust the Version and Provider if required, and then Next:

On the last page once again ensure that the Generate Sample Code checkbox is selected, and press Finish.

In the workbench the new project should have been created. Open up the Package Explorer to view its contents:

As you can see the project consists primarily of a single test, CustomerTest, and a set of configuration files. In the exec folder are three local launch configuration definitions to launch the application in one of two modes, or to run the tests. Let's look at those next.
