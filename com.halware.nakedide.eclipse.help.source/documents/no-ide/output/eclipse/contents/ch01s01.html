1. Create a new Application Project

Naked Objects applications are typically broken into several different projects, the most important of which is the applicaiton project. This contains the domain objects and supporting services that make up the application. The Application Project Wizard allows you to quickly create a simple 3-class sample application with the classpath and other resources correctly set up.

Use File>New>Project...>Naked Objects to list the NO IDE's project wizards. Select the Application Project wizard and press Next:

This brings up the first page of the wizard. In the Project name field, enter a name for the project, and change any other settings. Then press Next.

On the second page, adjust the Version and Provider if required, and then Next:

On the third and final page, ensure the Generate sample code checkbox is ticked, and press Finish:

The wizard copies example code and icons to create the new project. When it is done the application workbench will be displayed, showing the project:

There are a couple of things to note. First and most obviously is the layout of the workbench itself, which is as defined by the new Naked Objects perspective. This perspective brings together all the most useful views provided by the NO IDE. Second, the project has a Naked Objects nature (note the small 'N' overlay on the project); various other features of the NO IDE enable themselves based upon the presence of this nature. Third, note that the project that has has been created is in fact an OSGi (or plugin) project, hence the manifest in the editor area.

Let's now look at the generated code in the Package Explorer:

The application project includes a very simple (3 class) domain object model, along with 3 supporting repositories. These make a good starting point for any new application - just refactor them and modify them as required. Note also the provided images, with names matching the classes.

Although the application project is the most important of the projects to create, it isn't possible to run the client yet. For that we need both a fixture project and a client project.
