11. Running the Naked Objects Clients

The tutorial describes running the NOF Dnd client launch configuration and the Web local server launch configuration, but only against the in-memory persistor. However, for demos it can sometimes be useful to run as a standalone client and use the xml persistor instead:

This stores the objects in a series of XML files; the bottom line is that the objects are persisted from one run to the next.

The Naked Objects Framework also supports a Hibernate-based persistor for storing objects to RDBMS. This is not supported directly by the NO IDE, but can be used in conjunction with the client type (client/server mode) where the server is set up to use Hibernate. Consult the NOF documentation for further details.
