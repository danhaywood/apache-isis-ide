8. Extending the NO IDE

Currently the NO IDE offers only a very limited number of extension points. The intention is to increase these over time so that the NO IDE can be leveraged by any framework that implements the Naked Objects Architectural Pattern.

As well as other frameworks being able to contribute, it should also be possible to allow filtering/sorting of the default NO IDE functionality so that frameworks that offer a subset of the features supported by the NOF can remove stuff (such as annotations) that doesn't apply.
