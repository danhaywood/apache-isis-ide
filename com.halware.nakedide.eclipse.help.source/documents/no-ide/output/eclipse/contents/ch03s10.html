10. Format Code

Most developers get quite religious about code formatting standards - it suddenly seems to become very important where those curly braces go! While not wishing to be too dogmatic about these things, you will find that your domain objects can become quite heavy with annotations. These usually aren't too much of a problem with the one exception of parameters to actions. We therefore recommend that you adopt the standard of placing each parameter on a new line.
