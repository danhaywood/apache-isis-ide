8. Launch Configurations

The majority of the functionality in the NO IDE is generic for NOA-style applications, in particular against the Naked Objects application library. The launch configurations functionality however is specific to the Naked Objects Framework.

Use Run>Run...> or Debug>Debug... to bring up the launch configuration dialog:

The client project wizard automatically creates two launch configurations for the example project, one for DnD client and one for the Web Local Server (to access via a web browser). Either modify these or create new configurations as required.

On the Main tab, the main class is automatically specified (and cannot be modified). Since this class is in the NOF itself, the Include libraries when searching for a main class button is selected (and cannot be modified):

The most important tab however for Naked Objects Framework applications is the Arguments tab:

On this dialog:

  • the Viewer block corresponds to --viewer

  • the Type block corresponds to --type

  • the Authentication block corresponds to --user and --password

  • the Persistor block corresponds to --persistor

  • the Fixture block corresponds to --fixture

  • the Resource base block corresponds to --resourceBase.

  • the Program arguments block allows additional arguments to be specified

The options here enable and disable as required by the NOF runtime:

  • authentication details can be entered only for the dnd viewer, provided type is not exploration

  • the persistor cannot be entered for exploration or prototype (NOF uses an in-memory object store for these types)

  • the persistor is restricted to in-memory and xml when the type is standalone

  • the persistor is restricted to serialize and xstream when the type is client

  • the resource base can only be entered if the viewer is web.

One additional note about the resource base block (for the web viewer); this must refer to the base location for the images, usually held in the application project. The dialog attempts to guess this location based on the assumption that the launch configuration is being created in a client project whose name has a .client suffix.
