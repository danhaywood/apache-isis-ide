4. Apply Annotations

Many of the semantics for domain objects can be specified declaratively using annotations from the Naked Objects application library. The NO IDE allows these to be easily applied using a set of views:

  • the Naked Objects Properties, Collections and Actions view allows annotations to be specified on properties, collections and actions respectively

  • the Naked Objects String Properties view allows annotations specific to Strings (eg @MaxLength) to be specified for all properties whose type is java.lang.String

  • the Naked Object view allows annotations to be specified to the obejct itself.

As you enter values into these tables, the code is changed in real-time. Conversely, changing the code updates these views.

For all of these apart from the Naked Object view the table view (with many rows) makes sense because there can (of course) be many properties, collections or actions. The Naked Object view also uses a table even though there always be only be a single row in it:

On balance it was decided to implement the view this way for consistency with the other views.

The actions view is slightly more complex than the other views in that it also allows annotations to be applied to the parameters of each action:

Selecting on the different actions (on the left hand side of the view) brings up its parameters (if any) on the right hand side.

A few tips and tricks:

  • Each of the views has short-cut keys and are registered under the Window>Show View menu

  • Once a view has focus, use ctrl+M to maximize a view, and apply a set of annotations in one go.

  • Double clicking on a member name (property, collection or action) will select it in the code. Thus the views can be used as a simple way of navigating the domain object.

Please see the NOF documentation for further information on the detailed semantics of each of the annotations.
