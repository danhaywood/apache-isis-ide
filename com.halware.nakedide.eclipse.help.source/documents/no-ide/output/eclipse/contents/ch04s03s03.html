3.3. Service Templates

The service templates consist of two templates both relevant to repositories:

The findm template is used to return multiple instances from the object store, while the finds template returns the first instance. Each of these calls methods that exist in AbstractFactoryAndRepository, and so should only be used for repository implementations that subclass from this superclass. Both the templates also define an in-line implementation of the Filter interface, part of the Naked Objects application library. If the methods are performing the same filtering then it makes sense to pull this filter class definition out into a separate member (or even top-level) class.
