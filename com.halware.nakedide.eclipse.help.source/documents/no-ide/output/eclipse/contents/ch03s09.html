9. Update the Classpath

Because the projects created by the project wizards - and the Naked Objects runtime itself - are OSGi projects, this means that updating the classpath is performed by updating the project manifest (META-INF/MANIFEST.MF) rather than modifying the Java Build Path (from Project>Properties).

To make the changes, open up the Dependencies tab of the Manifest editor:

This corresponds to the Require-Bundle: entry in the MANIFEST.MF; alternatively you can modify this directly:

If you need to reference a 3rd party library - for example as part of the implementation of technical service - then you will need it as an OSGi bundle first. This may have been done for you, put if not it is not difficult to do.

  • There are a couple of OSGi bundle repositories to check. First, under Eclipse the orbit subproject has a small selection. Alternatively the osgibundles open source project, hosted on sourceforge and managed by the NOF/NO IDE developers (yours truly), has a reasonable collection. Contact us if you would like any additional JAR files uploading.

  • Otherwise though you can convert the 3rd party library yourself. Unless you want the capability to step into its source code, the easiest approach is to use Eclipse's wizard: File>New>Project...>Plug-in Development>Plug-in from existing JAR archives. All of the bundles in osgibundles have been built this way.
