2. Create a new Fixture Project

The role of a Fixture project is to hold fixtures that define a set of example objects. With these defined the client application can then be run. The Fixture Project Wizard sets up the structure of such a project, along with example fixtures for the Customer, Product and Order as created by the Application Project Wizard.

As before use File>New>Project...>Naked Objects to list the NO IDE's project wizards, but this time select the Fixture Project wizard and then press Next:

On the first page of the wizard, select the Application project that these are the fixtures for. This is used primarily to set up the classpath. Press Next:

On the second page of the wizard, enter the fixture project's Project name. This should have been derived from application project name given on the previous page. Then press Next:

On the third page, adjust the Version and Provider if required, and then Next:

On the last page again ensure that the Generate Sample Code checkbox is selected, and press Finish.

In the workbench the new project should have been created. Open up the Package Explorer to view its contents:

Each of these *Fixture classes uses the services of the *Repository classes in the application project to instantiate and persist new objects.
