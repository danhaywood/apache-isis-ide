2. Create a new Domain Object

Since NOA domain objects are very little more than POJOs, to create a new domain object simply use Eclipse's Java wizard: File>New>Class.... When done, though, use the following Domain Object templates:

  • the applib template (before the class definition) will quickly add the import statement for the Naked Objects application library

  • the entity template (again, before the class definition) will set up an @Entity declaration along with a comment.

  • the ident template can be used to create a title() method.

An icon is also required for the new class. The icon should be 32x32 pixels, can be a GIF, PNG or JPG, for example OrderItem.gif. For the NOF this icon file should reside in the application project under src/main/resources/images:

  • For the NOF DnD viewer this directory corresponds to a subpackage images on the classpath

  • For the NOF Web viewer this directory is referenced directly in the --resourceBase command line option (without the images suffix).

In addition, if the class is in a new package, then it will also be necessary to update the project manifest (META-INF/MANIFEST.MF) to export this package:

Navigate to the Runtime tab and click on the Add button under Exported Packages section. Alternatively, you can edit the Exported-Packages entry in the MANIFEST.MF itself:

This can sometimes be quicker than using the PDE manifest editor, but make sure that there is a carriage return after the last line, and the last line contains no characters.
