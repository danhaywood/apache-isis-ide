5.1. Problems Markers

Any problems located by the project builder are displayed in Eclipse's Problems view. The severity is always Warning (not Error):

The types of problems picked up by the project builder are:

  • Ambiguous method (could represent both a property/collection and an action

  • Incorrect parameter type for method (differs from accessor method return type)

    Applies to: mutator method, modify method, validate method,

  • Incorrect parameter type for collection method (should be parameterized type of the accessor method's return type)

    Applies to: addTo method, removeFrom method, validate method, validateAddTo method, validateRemoveFrom method,

  • Parameter types for actions are incorrect (should be same as action method's)

    Applies to: disable method, hide method, default method, choices method

  • Method for property should have zero parameters

    Applies to: clear method, disable method, hide method, default method, choices method,

  • Method for collection should have zero parameters

    Applies to: disable method, hide method,

  • Orphaned method (no corresponding accessor or action method)

    Applies to: mutator method, modify method, clear method, addTo method, removeFrom method, validate method, validateAddTo method, validateRemoveFrom method, disable method, hide method, default method, choices method,

  • validate prefix incorrect for collections - use validateAddTo or validateRemoveFrom

  • title method must return a String

  • No title or overridden toString
